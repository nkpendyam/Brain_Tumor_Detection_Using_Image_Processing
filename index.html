Step-by-Step Execution

Complete Pipeline Flowchart

Eight scripts, each with built-in skip logic. Already-completed stages auto-skip on re-run. Run in order: 01 → 01b → 02 → 03 → 04 → 05 → 07 → 06.

01
📥 Base Neuroimaging Data Acquisition 01_neuroimaging_data_acquisition.py
Downloads 2-D MRI and CT classification datasets from Kaggle. Routes images into semantic class folders inside dataset_ensemble/. Also downloads chest X-rays and facial photos as negative out-of-distribution samples for the Gatekeeper.

Classes collected: Glioma (1,621) · Meningioma (1,645) · NoTumor (3,500) · Pituitary (1,757) · Tumor_Generic (1,500)
✓ Already done (10,023 images) Kaggle API ~3.7 GB download
01b
🧠 Volumetric Patient Study Download 01b_volumetric_dataset_download.py
Downloads patient-level brain MRI studies for volumetric fine-tuning. Uses Kaggle only — no direct HTTP downloads that previously caused corruption. Auto-upgrades the Kaggle CLI at startup to fix the {current_version} download bug. Deletes stale partial downloads before each attempt using --force flag.

Extracts NIfTI volumes into per-patient JPEG slice folders (≤ 120 slices each) for training.
▶ Run if dataset_volumetric/ empty 🔧 Corruption bug fixed LGG 714 MB · Healthy ~200 MB
02
🚨 Safety Gatekeeper Training 02_gatekeeper_model_training.py
Fine-tunes EfficientNet-B0 as a binary Brain / NotBrain classifier. Trained on brain MRI/CT positives and chest X-ray + facial photo negatives. Runs 5 epochs with AdamW optimiser and AMP (Automatic Mixed Precision) on CUDA.

At inference: any slice scoring >70% NotBrain probability is silently rejected before reaching the Council — protecting diagnostic integrity from corrupted or irrelevant uploads.
✓ Done → Gatekeeper_v1.pth EfficientNet-B0 · 5 epochs · 224×224
03
🎯 Tumour Localizer Training (Hunter) 03_tumor_localization_model_training.py
Trains a YOLOv11n (Nano) single-class object detector to predict bounding boxes around tumour regions — telling clinicians where the tumour is, not just whether it exists. Uses Mosaic augmentation (50%) to force local feature attention. Exported to TorchScript for production deployment.

Mixup is intentionally disabled: blending anatomical regions creates clinically invalid training samples.
✓ Done → best.pt (mAP50 = 0.516) YOLOv11n · 35 epochs · 640×640
04
🏛️ Diagnostic Council Training 04_diagnostic_ensemble_training.py
Trains three independent 5-class classifiers in sequence. The final prediction is a weighted probability vote: P = 0.4·P_swin + 0.3·P_conv + 0.3·P_monai

Class-weighted cross-entropy handles the dataset imbalance. Each branch saves its best checkpoint by validation macro F1. All components are inlined — no external shared module needed.
🌐
SwinV2-Tiny
Swin_5C.pth
40% vote
Window-shifted attention captures global brain texture and long-range spatial relationships across the full MRI slice.
⚡
ConvNeXtV2-Nano
ConvNext_5C.pth
30% vote
Modern convolutional design provides rotation invariance and spatial stability — important for varied MRI acquisition orientations.
🏥
MONAI Swin-UNETR
MONAI_5C.pth
30% vote
Medical-domain pretrained encoder with anatomical intuition. Uniquely supports Grad-CAM saliency map generation for explainability.
✓ Done → 98.25% accuracy, F1=0.9783 12 epochs · 256×256 · AdamW + Cosine LR
05
🔬 Volumetric Fine-Tuning 05_volumetric_brain_finetune.py
Adapts the trained Council to full patient studies using patient-level train/val splits — preventing slice-level data leakage. All three backbones are frozen; only classification heads are updated for 4 epochs. Handles NIfTI volumes, DICOM series, and image folders.

Uses SHA-256 dataset fingerprinting (Volumetric_Finetune.json) to skip retraining if the dataset hasn't changed since the last run.
▶ Run after 01b 4 epochs · frozen backbones · head-only tuning NIfTI · DICOM · image folders
07
📊 Ensemble Performance Evaluation 07_ensemble_performance_evaluation.py
Zero-training evaluation on the same 20% holdout (random_state=42) used during council training. Reports per-class precision, recall, F1-score, support, overall accuracy, and macro F1. Saves a full confusion matrix to BrainTumor_Confusion_Matrix.csv.
Same val split as Script 04 BrainTumor_Confusion_Matrix.csv
06
🖥️ Clinical Dashboard 06_clinical_diagnostic_interface.py
Gradio web application on port 7860. Accepts DICOM series, NIfTI volumes, and MRI/CT images. Runs the full pipeline per-slice: Gatekeeper → Council vote → study-level aggregation. Generates a Grad-CAM saliency overlay on the highest tumour-signal slice and exports a formatted PDF clinical report.

Slice count is always shown clearly: "1 slice uploaded and analysed" vs "N slices uploaded — M analysed (uniform sampling)".
▶ Final step → http://localhost:7860 🔧 fpdf2 XPos/YPos · nibabel fix · no HYDRA

Tech Stack

Technologies Used & Why

Every library was chosen for a specific reason. This section explains each technology, how it fits the system, and what problem it solves.

🔥Core
PyTorch
≥ 2.5.0 · CUDA 12.8 (cu128)
The deep learning framework powering all model training and inference. CUDA 12.8 wheels are required specifically for the RTX 5060 (Blackwell SM_120 architecture) which is not supported by older torch builds.
RTX 5060 requires cu128 wheels; torch.amp (non-deprecated) for AMP; supports all five model architectures used in this project.
🌐Council
timm (SwinV2 + ConvNeXt)
≥ 1.0.0
PyTorch Image Models library providing pretrained SwinV2-Tiny and ConvNeXtV2-Nano weights. Both models are loaded with ImageNet pretraining, then fine-tuned on the brain tumour dataset for 12 epochs.
Single unified API for two radically different architectures. SwinV2 excels at global texture; ConvNeXt excels at local spatial patterns — complementary strengths for ensemble diversity.
🏥Council
MONAI
≥ 1.3.0 + einops
Medical Open Network for AI. Used for the Swin-UNETR backbone (medical imaging pretrained), and the GradCAM implementation that generates the saliency heatmaps on the peak tumour-signal slice.
Only library with a Swin-UNETR pretrained on medical images. Its GradCAM implementation works natively with the MONAI Swin-UNETR backbone for explainability without custom layer hooks.
🎯Localizer
Ultralytics (YOLOv11)
≥ 8.3.0
Used for YOLOv11n — the nano variant of YOLOv11 for real-time tumour bounding box prediction. Single-pass detection + bounding-box regression with Feature Pyramid Network for multi-scale detection.
YOLOv11n is the fastest model that still achieves clinically useful mAP50 (0.516). Mixup disabled because blending anatomical regions creates medically invalid training samples.
🎨UI
Gradio
≥ 4.42.0
Web-based AI application framework. Powers the clinical dashboard on port 7860. Handles multi-file upload (DICOM series, NIfTI, images), real-time progress display, and file download for PDF reports.
Zero-JavaScript required for a full-featured AI demo interface. Native file upload supports multiple files simultaneously — required for DICOM series upload where each slice is a separate file.
🧠Imaging
nibabel
≥ 5.2.0
Neuroimaging file I/O library for reading NIfTI volumes (.nii, .nii.gz). Used to load volumetric MRI data and extract 2-D axial slices for classification, trimming 15% from each end to avoid skull/noise regions.
The only reliable Python library for NIfTI format. Used via nibabel.loadsave.load() (not nib.load()) to avoid a Pylance reportPrivateImportUsage warning on strict type checking.
🏥Imaging
pydicom
≥ 2.4.0
DICOM file reader. Used to parse individual .dcm files, sort them by ImagePositionPatient Z-coordinate (or InstanceNumber fallback) to reconstruct the correct axial ordering of a scan series before inference.
DICOM files have no guaranteed filename ordering — proper sorting by Z-position is critical to avoid giving the model slices in the wrong order, which would degrade temporal feature coherence.
👁️Vision
OpenCV + Pillow
cv2 ≥ 4.9 · Pillow ≥ 10
OpenCV handles skull-stripping (Otsu thresholding + largest contour), colour map generation for Grad-CAM overlays, and BGR→RGB conversion. Pillow is used for all PIL Image operations and format loading.
OpenCV's contour detection is the fastest approach for best-effort skull stripping without a trained segmentation model. Pillow's ImageOps.mirror() provides test-time augmentation with zero latency.
📄Reports
fpdf2
≥ 2.7.0
Pure Python PDF generation library. Used to produce the downloadable clinical report including the finding verdict, confidence, probability table, branch notes, and Grad-CAM saliency image.
Lightweight, no external dependencies. Uses the new new_x=XPos.LMARGIN, new_y=YPos.NEXT API (not deprecated ln=True) to avoid TypeError on fpdf2 ≥ 2.7.
📊Training
scikit-learn
≥ 1.4.0
Used for stratified train/val splitting, class-weight computation (inverse-frequency balancing), classification report, confusion matrix, accuracy, and macro F1 calculation throughout all training scripts.
compute_class_weight("balanced") automatically corrects the class imbalance in the dataset (No Tumor is over-represented). Stratified split ensures each class appears in validation proportionally.
🐋Deploy
Docker + NVIDIA Container Toolkit
CUDA 12.8 base image
Containerised deployment using nvidia/cuda:12.8.0-cudnn9-runtime-ubuntu22.04 base. The full system including Python 3.11, all dependencies, and the Gradio dashboard can be deployed with a single docker compose up.
Reproducible environment that works across any CUDA 12.8 compatible machine. Weights and datasets are mounted as volumes (not baked into the image) to keep the image size manageable.
📡Data
Kaggle API
≥ 1.6.0 (auto-upgraded)
All training data is downloaded programmatically via the Kaggle CLI. The download script auto-upgrades the CLI at startup to fix the {current_version} placeholder bug that caused corrupted ZIP downloads, and uses --force to prevent stale partial file resumption.
Reproducible data acquisition — any machine with a Kaggle account can recreate the full dataset. The --force flag is critical: without it, partial downloads from previous failed runs corrupt the ZIP.
